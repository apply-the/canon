## Phases

### Phase 0: Version Bump (TASK-001)

Bump `version` in workspace `Cargo.toml` from `0.52.0` to `0.53.0`.

### Phase 1: Domain Model (TASK-002 — TASK-003)

1. Add `ConventionalCommentScope` enum in `findings.rs` with variants `Pr`, `File`, `Surface`, serde `rename_all = "kebab-case"`, strum `Display + IntoStaticStr`.
2. Extend `ReviewFinding` with `scope: ConventionalCommentScope`.
3. Implement `derive_scope(changed_surfaces, category)` private helper in `ReviewPacket::from_diff` construction that:
   - returns `Pr` when `changed_surfaces` is empty,
   - returns `Surface` when all surfaces share a single classification heuristic (all test, all contract, all boundary),
   - returns `File` otherwise.
4. Unit tests for all scope derivation cases.

### Phase 2: Renderer Update (TASK-004)

Update `render_conventional_comments(packet)` in `markdown.rs` to include the scope annotation in each entry. Update the `conventional-comments.md` renderer branch inside `render_pr_review_artifact` to reflect the evidence posture text for scope.

### Phase 3: Skill Documentation (TASK-005)

Update `canon-pr-review` SKILL.md to document the three scope levels and the deterministic derivation rule.

### Phase 4: Validation (TASK-006)

- `cargo fmt --check`
- `cargo clippy --workspace --all-targets --all-features -- -D warnings`
- `cargo nextest run`
- `cargo llvm-cov --workspace --all-features --lcov --output-path lcov.info` and verify 95% line coverage on `findings.rs` and `markdown.rs` (only modified lines).
