# Roadmap

Welcome to the official **Canon** Roadmap. Here we track the evolution of our *Semantic Governance* engine for engineering work.

::: info Vision
The goal of Canon is to transform AI-assisted engineering activity into a process based on governed packets, irrefutable evidence, and a durable Project Memory.
:::

## <i class="fa-solid fa-rocket"></i> Upcoming Features & Topics

### Current 0.63.1
- **Guided `pr-review` Ref Choice**: Canon 0.63.1 now hardens `canon-pr-review` intake with a guided comparison choice when base/head refs are missing or semantically unclear, while preserving host-rich-input support and plain guided-text fallback.

### Recently Landed
- **Assistant Surface Expansion**: `canon init` now supports Cursor and Antigravity alongside the existing assistant targets, while repository-level assistant package metadata remains aligned with the current release.

### Onboarding & Integrations
- **External Publish & Backlog Destinations**: Capturing operator choices for publishing packets and handoff destinations.
- **MCP-Backed Credential References**: Securely integrating with external systems like Atlassian-family tracking without persisting raw tokens in versioned state.
- **Extended Init Integrations**: Building on the shipped guided init flow to capture optional publish, backlog, or integration surfaces without weakening the non-interactive contract.

### Core Philosophy
This roadmap remains intentionally sparse: a macrofeature only moves forward once its bounds, artifact contract, and validation story are explicit.
